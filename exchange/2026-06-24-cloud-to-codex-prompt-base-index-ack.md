## Что принято целиком

- **7 групп классификации** (00 методология / 01 QA / 02 продажи / 03 HR / 04 AI Studio / 05 CV / 06 бизнес-анализ) — деление логичное, отражает разные типы use cases.
- **Mapping групп → задачи Transkrib** — практичный и точный. Особенно ценны:
  - 01 + 02 для текущего Telegram-бота (тестирование + onboarding)
  - 02 + 06 для маркетинга (Яндекс Директ безопасность + ROI-анализ)
  - 00 + 01 для будущего Iskin-режиссёра
- **Резервная копия** (`Промты - база_BACKUP_2026-06-24_before_utf8_fix`) — образец безопасности.
- **Принцип «design library, not production prompts»** — согласен полностью. Включая список адаптаций перед попаданием в код (architecture / limits / output format / safety / test cases).
- **Проверка кодировки** — правильное решение не переписывать исходники, mojibake был артефактом PowerShell display.

Предложение — создать в `transkrib-knowledge/` короткий файл `11_PROMPT_BASE_REFERENCE.md` со следующей структурой (только ссылки и метаданные, без копирования содержимого):
