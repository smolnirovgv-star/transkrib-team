# transkrib-team

**Публичное координационное пространство** команды координации проекта Transkrib.

---

## Команда

| Участник | Роль | Сильная сторона |
|---|---|---|
| **Геннадий** | Owner проекта, финальное слово (~34% голоса в коллегиальных решениях) | Стратегия, бизнес-решения |
| **Cloud** (Claude) | Оператор/стратег старого стека; консультант нового | Работа в браузере, аудит, документация |
| **Codex** | Оператор нового стека (AI Router, multi-agent); консультант старого | Прямой доступ к файловой системе |
| **Cursor** | Кодировщик в обеих зонах | Код, git, file edits |

Решения принимаются **коллегиально**. Финальное слово — за Геннадием.

---

## Зачем этот репо

Между сессиями участники команды не имеют общей памяти. Этот репо — **единый источник правды** о текущем состоянии проекта, который читают все участники команды:

- **Cloud** читает через `web_fetch` напрямую (репо публичный)
- **Codex** и **Cursor** читают через `git clone` / `git pull`
- **Геннадий** читает в браузере на github.com

Это снимает трение «копировать-вставлять контекст между чатами» и ускоряет старт каждой новой сессии.

## Структура

| Файл / папка | Назначение |
|---|---|
| `README.md` | Этот файл |
| `HANDOFF_CURRENT.md` | Актуальное состояние проекта — обновляется в конце каждой сессии |
| `DECISIONS.md` | Лог архитектурных и командных решений |
| `OPSEC_RULES.md` | Правила: что можно публиковать в этот репо, что нельзя |
| `reviews/` | Обмен ревью между агентами (например, `CODEX_CLOUD_REVIEW.md`, `CLOUD_RESPONSE.md`) |
| `Issues` | Открытые вопросы, баги, задачи. Закрываются PR или коммитом. |

---

## Правила работы

1. **Перед коммитом в этот репо** — проверить файл по `OPSEC_RULES.md`. Если есть сомнения, что что-то чувствительное может попасть наружу — **класть в приватную папку** `C:\Dev\Codex plus Cloud\` локально, не сюда.
2. **`HANDOFF_CURRENT.md` обновляет тот, кто завершил значимый блок работы** (Cloud, Codex или Геннадий). Файл должен быть актуальным на начало каждой новой сессии.
3. **Кодировка всех `.md` файлов — UTF-8 с BOM**. Это устраняет проблему с кракозябрами на Windows.
4. **Issues именуются префиксом**: `[bug]`, `[discussion]`, `[task]`, `[architecture]`.
5. **Все commit messages — на английском**, чтобы не зависеть от кодировки терминалов.

## Что НЕ публикуется здесь

Подробный список в `OPSEC_RULES.md`. Кратко:

- API-ключи, токены, пароли, секреты (ни в каком виде)
- Service IDs Railway, Render, Supabase, и других внутренних идентификаторов инфраструктуры
- ShopID платёжных систем (ЮKassa)
- Конкретные финансовые цифры (выручка, баланс, расходы)
- Детали уязвимостей в живом коде с эксплоит-информацией
- Персональные данные клиентов и бета-тестеров

Эти вещи живут в приватной базе знаний `transkrib-knowledge` (отдельный приватный репо) и в локальной общей папке `C:\Dev\Codex plus Cloud\`.
